1. Поднять web-сервер на raspberry pi с белым ip-адресом.
При подключении к серверу у пользователя отображается веб-страница с кнопкой. 
При нажатии на кнопку сервер отсылает json пакет на клиентскую raspberry pi. 
Получив из сети пакет, распберри клиента по spi интерфейсу отправляет пакет из 2х байт на драйвер, управляющий двигателями.

-----
1. Есть распберри, с белым Ip-адресом, доступом по ssh и открытым портом 5000 для работы сервера.

Задача 1.1:
Подключиться к удаленной распберри по ssh.
Поднять на ней сервер с отображением web-страницы с кнопкой. 
При нажатии на кнопку, расберри-сервер отправляет json-пакет на распберри клиента.
Обмен пакетами происходит через веб-сокеты.
Распберри клиента выводит в терминал лог, с принятым пакетом.

Задача 1.2:
Поднять spi-интерфейс на распберри.
Замкнуть  выход со входом spi-интерфейса (MOSI и MISO).
Отправить пакет по spi, вывести в лог терминала принятый со входа пакет.

Задача 1.3:
Поднять на микроконтроллере spi-интерфейс (stm32 или ардуино). 
Принять spi-пакет от распберри-клиента.
Вывести по уарту лог с полученным пакетом.

Задача 1.4:
Поднять spi-интерфейс на распберри клиенте.
При получении пакета от распберри-сервера, распберри-клиент отаправляет по spi пакет из двух байт.
